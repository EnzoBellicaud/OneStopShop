OneStopShop — Scraper & Import Architecture

Two decoupled APScheduler jobs share a CrawlUrl queue table. The Crawler discovers URLs; the Scraper processes them. Manual imports write directly via the API.

Reflects logic in backend/content/scrapers/service.py, source_registry.py, and content/ingestion/importer.py

Job 1 — Crawler · every 360 min

URL Discovery → CrawlUrl Queue

Runs for each configured source. Populates the queue; never fetches or parses page content.

  A(["Crawler triggered"]) --> B["Load enabled sources"]

  D --> E{"crawl_enabled?"}

  E -- "Yes - BFS mode" --> F["Fetch seed URL HTML"]
  F --> G["Extract depth-1 links filtered by include/exclude patterns"]
  G --> H["get_or_create CrawlUrl per link — archived not resurrected"]

  E -- "No - single URL" --> I["get_or_create CrawlUrl for known URL"]

  H --> J{"More sources?"}
  I --> J
  J -- Yes --> D
  J -- No --> Z2(["Crawler done — queue updated"])

Job 2 — Scraper · every 5 min

URL Processing → Offer Upsert

Claims a batch of pending/due CrawlUrl rows and processes each one. Creates a ScrapingRun record with a full structured log.

flowchart TD
  A(["Scraper triggered"]) --> B["Claim up to BATCH_SIZE pending/due CrawlUrls"]
  B --> C{"URLs claimed?"}
  C -- No --> Z1(["Done — queue empty"])
  C -- Yes --> D["Create ScrapingRun — status RUNNING"]
  D --> E["For each URL"]

  E --> F["Fetch HTML"]
  F --> G{"HTTP result?"}

  G -- "404 / 410" --> P1["status = archived"]
  P1 --> P2["Archive or delete linked offer"]
  P2 --> LOG

  G -- "5xx / timeout" --> Q1["Increment consecutive_errors"]
  Q1 --> Q2{"errors >= MAX?"}
  Q2 -- Yes --> Q3["status = archived — archive offer"]
  Q2 -- No  --> Q4["status = error — backoff 1h / 6h / 24h"]
  Q3 --> LOG
  Q4 --> LOG

  G -- "2xx" --> H["Run deterministic extraction"]
  H --> I{"crawl_enabled source?"}

  I -- Yes --> J["AI relevance check and extraction via Ollama"]
  J --> K{"Relevant offer? Confidence >= threshold?"}
  K -- No  --> K1["Decision: NEGLECT — log reason code"]
  K1 --> K2["status = done — skipped"]
  K2 --> LOG
  K -- Yes --> L["Pick best payload: LLM if confidence >= deterministic"]
  L --> M["Upsert Offer — link CrawlUrl.offer"]
  M --> N["status = done — next_check_at = now + 7 days"]
  N --> LOG

  I -- No --> R["AI is primary — deterministic is fallback"]
  R --> L

  LOG["Append url_processed or url_neglected to ScrapingRun log"] --> MORE{"More URLs?"}
  MORE -- Yes --> E
  MORE -- No --> FIN["Update ScrapingRun counters and status"]
  FIN --> STALE["Flag stale candidates — successful sources only"]
  STALE --> Z2(["ScrapingRun complete"])

API — Manual Import

CSV / Excel → Offer + CrawlUrl

Operator uploads a file, reviews validation output, then confirms. No scraper job involved.

flowchart LR
  A(["Operator"]) --> B["GET /api/offers/import/template — download .xlsx"]
  A --> C["POST /api/offers/import/preview — upload CSV or Excel"]
  C --> D["Parse and validate rows — required fields, URL format, lookup values"]
  D --> E["Return PreviewResult — valid list and invalid list with errors"]
  E --> F{"Review and confirm"}
  F -- "POST confirm with rows" --> G["POST /api/offers/import/confirm"]
  G --> H["For each row: get_or_create Offer — status draft or published"]
  H --> I["Create CrawlUrl — source_key = import__ plus org_id — status pending"]
  I --> J(["ConfirmResult — drafts and published counts"])

CrawlUrl Status Lifecycle

stateDiagram-v2
  [*] --> pending : Crawler discovers URL or import confirms row

  pending --> processing : Scraper claims batch

  processing --> done     : 2xx — offer upserted or page skipped — next check in 7 days
  processing --> error    : 5xx or timeout — consecutive errors below MAX — backoff applied
  processing --> archived : 404 or 410 or consecutive errors reached MAX — offer archived

  error --> pending : Backoff expires — re-queued for retry

  done --> pending : Revisit window expires after 7 days
Decision codes logged per URL in ScrapingRun.log:
mapped — offer upserted successfully
neglected / non_relevant_page — crawl page rejected by AI
neglected / missing_core_fields — extraction returned incomplete data
failed — HTTP or network error
archived — permanent 404/410 or repeated errors exceeded limit
